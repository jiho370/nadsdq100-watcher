# 시스템 종합 정리 (SYSTEM_OVERVIEW.md)

이 문서는 이 저장소가 무엇을 하는 시스템인지, 알고리즘이 정확히 어떤 수치·규칙으로 동작하는지,
그리고 그 결과가 어떻게 매일 자동으로 메일까지 도달하는지를 한 문서에 정리한 것이다.
전략 수치의 1차 출처는 `STRATEGY.md`, 유지보수 변경 이력은 `HANDOFF.md`이며, 이 문서는 그 둘과
코드를 종합해 "처음 보는 사람도 전체 그림 + 재현 가능한 수치"를 파악할 수 있게 만든 요약본이다.

---

## 1. 이 시스템이 하는 일 (한 줄 요약)

매일 아침(한국장 개장 전)과 늦은 오후(미국장 마감 후) 두 통, 일요일 아침 한 통, 총 세 종류의
이메일을 자동으로 만들어 발송한다. 종목 선정과 매수/매도 실행 계획은 전부 **코드(규칙)가 확정**하고,
AI(Claude)는 최신 뉴스 검증과 서술(설명 문장)만 담당한다 — 즉 "무엇을 살지"와 "얼마에 얼마나
나눠 살지"는 100% 결정론적이고 재현 가능하며, AI가 바꿀 수 없다.

| 메일 | 발송 시각(KST) | 대상 요일 | 핵심 내용 |
|---|---|---|---|
| 🇰🇷 한국장 장전 | 08:00 | 월~금 | 전일 한국장 마감 데이터 기준 코스피200 매수/관찰/매도 + 밤사이 세계시장 요약 + 추세신호(코스피·코스닥·금) |
| 🇺🇸 미국장 마감 | 17:00경(16:40 실행) | 화~토 | 그날 새벽 마감된 미국장 기준 S&P500 매수/관찰/매도 + 추세신호(나스닥100·S&P500·비트코인) |
| 📊 주간 자산배분 | 07:30 | 일 | 표준 자산배분(안정형/공격형) + 리밸런싱 밴드 + 방어 컷 안내 |

핵심 철학(STRATEGY.md 서두): **추세 신호는 '더 벌기 위한 도구'가 아니라 '크게 잃지 않기 위한
도구'다.** 기대값이 높은 진입은 (1) 상승 레짐 확인 후 (2) 눌림에서 분할로 사고, (3) 정해진 이탈
기준에서 기계적으로 나온다는 원칙을 모든 규칙이 공유한다.

**두 개의 실행 경로**가 이 파이프라인을 돈다:

1. **로컬 PC 경로** (`pregen.py`) — Windows 작업 스케줄러가 밤/아침에 로컬 `claude -p`(Pro 구독,
   과금 없음)를 돌려 위 파이프라인의 검증+서술 단계를 **미리** 실행해 결과를 JSON 캐시로 저장.
2. **GitHub Actions 경로** (`daily_ai_report.py`) — 실제 발송 시각에 클라우드 러너가 항상 실행.
   위 캐시(pregen)가 있으면 그대로 재사용(AI 호출 0회에 가까움), 없으면(PC 꺼짐 등) 그 자리에서
   Anthropic API로 검증+서술을 전부 수행(폴백, 유료지만 발송은 절대 끊기지 않음).

## 3. 알고리즘 상세 (수치 전부 — 백테스트/재현용)

### 3.1 연구 근거 요약 (STRATEGY.md §0)

시스템에 들어간 모든 규칙은 아래 표의 문헌·자체 백테스트 근거를 통과한 것만 채택했다. 통계적으로
약하다고 판단된 것(200일선 타이밍의 초과수익, 골든크로스 개별종목 신호, "1주 ±5% 등락" 트리거)은
**폐기**했다.

| 주장 | 근거 수준 | 결론 |
|---|---|---|
| 200일선 위/아래 지수 타이밍 → 초과수익 | 약함(Zakamulin 표본외 검증에서 유의하지 않음) | 수익 목적 채택 안 함 |
| 200일선 타이밍 → MDD 대폭 축소 | 강함(Zakamulin, Faber 2007: MDD 절반 이하) | **채택 — 위험관리 장치로만** |
| 골든크로스(50/200) 개별종목 진입신호 | 약함(개별주 거짓신호 급증, 자체 백테스트 IC≈0) | 불채택(지수 레짐 설명용만) |
| 12-1 모멘텀(최근 1개월 제외 12개월 수익률) | 강함(Jegadeesh-Titman 1993 이후 국제 재현) | **채택 — 절대 모멘텀 필터** |
| 52주 고점 근접도 | 강함(George-Hwang 2004, 국제 20개 시장 중 18개 유효) | **채택 — 개별주 진입 필터** |
| 한국 시장 모멘텀 | 조건부(전체 유니버스는 무의미, 코스피200 한정 시 유의) | **채택 — 유니버스를 코스피200으로 제한** |
| 트레일링 스톱 | 조건부 강함(Dai 2021 등: 15~20%가 최적 구간) | **채택 — -20% 트레일링**(기존 -25%에서 강화) |
| 코인 추세추종 | 강함(BTC/ETH 매수후보유 MDD -83%/-94%) | **채택 — 120일선·3개월 모멘텀(더 빠른 창)** |
| 변동성 타깃팅 | 강함(Moreira-Muir 2017, Barroso-Santa-Clara 2015) | **채택 — 참고 지표로만 표기** |
| 리밸런싱 밴드(목표 대비 ±20%) | 양호(Daryanani 2008: 캘린더 대비 개선) | **채택 — 주간 배분의 매매 트리거** |

### 3.2 미국 개별주 선정 — S&P500

**팩터 가중치(백테스트로 확정, `output/best_weights.json`)**:

```
gross_margin × 2 + accruals × 2 + mom6 × 2 + mom12_1 × 1 + shareholder_yield × 1
```

- 팩터별 z-score(±3 클리핑)를 위 가중치로 합성해 랭킹(`export_data.select_by_weights`).
- **백테스트 성과**: 12개월 초과수익 **+20%p**, 승률 **89.5%**, 최악낙폭(MDD) **-1.8%**, 샤프 **1.6**,
  권장 보유기간 1개월.
- **워크포워드 검증**: 학습구간(IS) 6개월 +9.73%p → 표본외(OOS) +9.88%p (과최적화 아님 — OOS가
  IS보다 오히려 높음).
- **탈락한 팩터**(검증 후 배제): 골든크로스류(IC≈0), 저변동성, 단기 반전, 오버나이트 모멘텀
  (IC -0.017, 대형주에서 역효과). 잔차 모멘텀은 약함(IC 0.004).
- **채택된 개별 팩터 IC**(예측력 순): gp_assets(0.088) · gross_margin(0.075) · shareholder_yield ·
  accruals · roa · mom6 · cop(0.029).
- 재검증 명령: `python backtest_weights.py --years 10 --keep 8 --oos 0.4`
  (IC 상위 8개 팩터로 가중치 재탐색, 표본외 40%로 워크포워드 검증. 결과는
  `output/best_weights.json`/`ic_report.json`에 저장 — 없으면 12-1 모멘텀 단일팩터로 자동 폴백).
- **재검증 주기 권장**: 분기~반기(팩터 IC는 시간이 지나며 감쇠하는 경향).

**진입 필터**(`export_data.split_by_entry` — '지금 매수'에 오르기 위한 추가 조건):
1. 종가 > 200일선
2. 52주 고점 대비 **-25% 이내**(George-Hwang 근접도 — 너무 먼 종목은 모멘텀 신뢰도 낮음)
3. 과열 판정: **RSI ≥ 72** 또는 **50일선 이격 +15% 이상** → '분할 매수' 강제 표기(hot=True)

조정 중(눌림목→관찰) 판정: 1주 수익률 **-2% 이하** 이거나 종가가 **20일선 대비 -2% 초과 이탈**.

**매도 규칙**(`holdings.py`, 둘 중 먼저 걸리는 것):
- 트레일링 스톱: 보유 중 고점 대비 **-20%**
- 추세 이탈: 종가 < 200일선 × **0.97**(3% 버퍼)

**후보풀 → 최종 채택 규모**: 후보 풀 6~7종목(`REPORT_POOL`, pregen은 버퍼 +3) → AI 검증 후
최종 매수 **4** · 관찰 **4**(`REPORT_FINAL_BUY`/`REPORT_FINAL_WATCH`, 카드가 상세해지며 5→4로 축소).

### 3.3 한국 개별주 선정 — 코스피200 (`kr_stocks.py`)

- **유니버스**: 코스피200 구성종목만(전체 상장사 유니버스에서는 모멘텀이 무의미하다는
  universe-shrinkage 연구 근거 — arXiv:1211.6517).
- **펀더멘탈 필터**(KRX 공식 데이터, pykrx):
  - EPS > 0 (흑자)
  - ROE(=EPS/BPS) ≥ **8%** (`KR_ROE_MIN`)
  - 0 < PER ≤ **40** (`KR_PER_MAX`)
- **추세 랭킹**(필터 통과 종목 중, 종가 > 200일선 필수):
  ```
  점수 = z(12-1 모멘텀) × 0.6 + z(52주 고점 근접도) × 0.4
  ```
- **눌림목(관찰) 판정**: 20일선 대비 -2% 초과 이탈 또는 1주 수익률 -2% 이하 (미국과 동일 기준).
- **후보풀 → 최종**: 풀 매수 4·관찰 3(`KR_POOL_BUY`/`KR_POOL_WATCH`) → AI 검증 후 최종
  매수 **3** · 관찰 **2**(`KR_FINAL_BUY`/`KR_FINAL_WATCH`).
- **매도**: 미국과 동일 규칙(-20% 트레일링 / 200일선 -3% 이탈), `output/kr_holdings.json`으로
  추천 이력 자동 추적.
- **데이터 의존성(중요)**: 2025-12-27부터 KRX 정보데이터시스템이 로그인 필수(KRX Data
  Marketplace)로 바뀌어 `KRX_ID`/`KRX_PW` 환경변수가 없으면 이 섹션 전체가 비어서 발송된다
  (에러는 아님 — §6.4 참고).

### 3.4 지수·코인·금 추세신호 엔진 (`market_signals.py`)

6개 핵심 자산을 레짐(추세 on/off) + 절대 모멘텀으로 5단계 신호를 매긴다. **완전히 stateless**
(종가 시계열만으로 상태를 매번 복원하므로 상태파일이 필요 없다).

| 구분 | 추세선 | 히스테리시스 밴드 | 확인 일수 | 절대 모멘텀 | 눌림 기준선 | 변동성 타깃 |
|---|---|---|---|---|---|---|
| 주식형(코스피·코스닥·금·나스닥100·S&P500) | 200일선 | ±1% | 3거래일 | 12-1 수익률 > 0 | 20일선 | 연 15% |
| 코인(비트코인) | 120일선 | ±3% | 3일 | 3개월 수익률 > 0 | 50일선 | 연 40% |

- **히스테리시스 규칙**: 추세선 +밴드 위로 3일 연속 마감 → 레짐 ON. -밴드 아래로 3일 연속 →
  레짐 OFF. 밴드 안(중간)에서는 직전 상태 유지 — 거짓신호(whipsaw) 억제가 목적.
- **변동성 타깃 노출**: `W = min(1, 타깃변동성 / 최근 60일 실현변동성)` — 참고 지표로만 표기(매매
  신호는 아님).

**5단계 상태 판정표**:

| 상태 | 조건(레짐×모멘텀) | 행동 |
|---|---|---|
| 🟢 적극 매수 | ON + 모멘텀+ (눌림선 위) | 정기 적립 계속 + 신규 매수 가능 |
| 🔵 눌림목 분할 매수 | ON + 모멘텀+ (눌림선 아래) | **기대값 최상 구간** — 2~3회 분할 매수 |
| 🟡 보유 | ON + 모멘텀− | 기존 보유 유지, 신규 보류 |
| 🟠 축소 검토 | OFF + 모멘텀+ | 신규 중단, 반등 시 일부 축소 |
| 🔴 위험 회피 | OFF + 모멘텀− | 신규 중단 + 비중 절반 이상 축소 |

### 3.5 실행 계획(분할매수·손절·처분) — `entry_plan.py`, 100% 하드코딩

"일관된 패턴은 하드코딩, 변하는 것만 AI" 원칙에 따라 이 부분은 **AI가 절대 못 바꾸는 값**이다.

**매수 계획**:
- 평시(과열 아님): **2분할** — 1차 50% 현재가 / 2차 50% 20일선 부근(현재가보다 위면 현재가의
  -3% 지점으로 대체).
- 과열(hot=True, RSI≥72 또는 50일선 +15%↑): **3분할** — 1차 30% 현재가 / 2차 30% 20일선 부근 /
  3차 40% 50일선 부근.
- 하한선: 분할 가격이 200일선 아래로 내려가면 200일선까지로 올림(추세 이탈 구간 매수 금지).
- **손절선** = `max(1차 매수가 × (1 - 0.20), 200일선 × (1 - 0.03))` — 매수 직후 기준(보유 후엔
  holdings.py의 트레일링 -20%가 이어받음).

**관찰 → 매수 전환 조건**: 20일선 회복(또는 위 유지 시 1주 수익률 플러스 전환), RSI ≤ 70,
전환 시 계획은 '평시 2분할'과 동일.

**매도(처분) 계획**:
- 손실이 **-15% 초과**면 전량 즉시 정리(추가 하락 방어).
- 그 외에는 50% 즉시 정리 + 나머지 50%는 반등 시 20일선 부근에서 정리(2주 내 미반등이면 전량).

가격 단위: 미국(USD)은 소수 2자리, 한국(KRW)은 정수 원.

### 3.6 주간 자산배분 (`weekly_report.py`, 일요일 전용)

**표준 배분**(통념·연구 부합, 기존 '금 25~30%' 배분은 과최적화 판단으로 폐기):

| 자산군 | 안정형 | 공격형 | 근거 |
|---|---|---|---|
| 미국 주식 | 30 | 50 | 전통 60/40 틀 |
| 한국 주식 | 10 | 15 | 자국 편향 최소화, 환헤지 성격 |
| 채권 | 40 | 15 | 위기 시 완충(주식과 낮은 상관) |
| 금 | 10 | 10 | 연구 권장 범위 5~15% 내 |
| 코인(BTC 중심) | 2 | 5 | MDD -80%대 자산 — 기관 가이드라인 1~5% 상한 |
| 현금성 | 8 | 5 | 리밸런싱 실탄 |

- **리밸런싱 밴드**(`REBALANCE_BAND = 0.20`, Daryanani 2008): 실제 비중이 목표의 **1.2배 초과**
  → 초과분만 매도(차익실현). 목표의 **0.8배 미만** → 부족분 매수(저점매수). 단 해당 자산 레짐
  OFF면 3회 분할로 진입.
- **방어 컷(추세 오버레이)**: 자산이 레짐 OFF **+ 12개월 수익률 음수**(`ret_1y < 0`) → 목표 비중의
  **절반만** 유지, 컷분은 채권·현금으로. 레짐 ON 복귀 시 원상복구. (일일 신호 엔진과 같은 규칙 —
  신호 일관성 유지.)
- 폐기된 구규칙: "1주 ±5% 등락 시 차익실현/저점매수 후보"(소음 기반, 검증 없음).

### 3.7 AI 검증 레이어 — 규칙과 AI의 역할 분담 (다시 강조)

- 규칙이 후보 '풀'을 뽑는다(미국 매수 4·관찰 4 + 버퍼, 한국 매수 4·관찰 3).
- AI(sonnet, 웹검색 ≤4회)가 각 후보를 검증해 `매수유지 / 관찰강등 / 제외` 판정.
- **AI는 종목을 추가할 수 없다**(할루시네이션 차단). 축소만 가능하며, 과도 제외로 매수가
  최소치(3개) 미만이 되면 강등분에서 점수순으로 복원한다(제외분은 복원 안 함).
- 최종 확정은 코드가 한다(`_apply_verdicts`) — 미국 최종 4/4, 한국 최종 3/2.
- 종목별 근거(points)는 항상 4축 고정 순서: **①추세 ②펀더멘털·사업 ③뉴스 ④촉매/리스크**.
- AI가 제외한 후보와 사유는 리포트에 별도 표기(투명성 — "왜 빠졌는지"도 판단 재료).
- 어떤 단계든 AI가 실패하면 지표 기반(`deterministic_report`)으로 무조건 발송 — **발송 누락은
  절대 없음**.

## 4. 비용 최소화 설계 (AI 호출을 어디까지 줄였는가)

초기 버전은 회당 **~$0.5**(월 10만원대)가 청구됐다. 지금은 아래 구조로 **pregen이 정상 작동하는
날은 발송 시점 API 호출이 사실상 0회**, 실패하는 날(PC 꺼짐 등)만 폴백 비용이 발생한다.

### 4.1 "일관 패턴은 하드코딩, 변하는 것만 AI"

| 내용 | 처리 방식 | 비용 |
|---|---|---|
| 종목 선정(팩터랭킹·필터) | 100% 코드 | $0 |
| 분할매수가·손절선·전환조건·처분계획 | 100% 코드(`entry_plan.py`) | $0 |
| 지수·코인·금 추세신호 | 100% 코드(`market_signals.py`) | $0 |
| 종목 사업설명(프로필) | AI **1회 생성 후 캐시 재사용**(`gen_profiles.py`) | 분기 1회, 로컬 CLI면 $0 |
| 최신 뉴스·악재 검증 | AI(sonnet + 웹검색, 후보 개수만큼 아님 — 묶음 검색) | pregen 있으면 $0, 없으면 폴백 |
| 종목별 서술(요약/포인트/코멘트) | AI(haiku, 검색 없음) | pregen 있으면 $0, 없으면 haiku 1콜 |

### 4.2 모델 분리(opus 사용 금지)

- `REPORT_MODEL_VERIFY = claude-sonnet-5` — 검증 단계(웹검색 필요, 판단력 필요).
- `REPORT_MODEL_WRITE = claude-haiku-4-5` — 서술 단계(사실이 이미 입력에 다 있음, 저가 모델로 충분).
- `_no_opus()` 함수가 코드 레벨에서 opus 계열 모델을 sonnet으로 강제 치환(비용 정책).
- API 경로(`_call_api`)·CLI 경로(`_call_cli`) 둘 다 동일하게 `model` 인자로 이 분리를 강제한다
  (2026-07-09 이전엔 CLI 경로에 model 인자가 아예 없어서 두 단계 모두 CLI 기본모델로 나가던 버그가
  있었음 — 수정 완료, §6.2 참고).

### 4.3 로컬 Pro 구독 CLI를 활용한 완전 사전생성 (`pregen.py`)

메일이 국장/미장 2통으로 분리되면서, 각 pregen 실행 시점에 **이미 해당 세션 데이터가 마감
확정**된다는 점을 이용해 검증뿐 아니라 서술까지 미리 만들어 캐시한다.

| pregen | 실행 시각(KST) | 유효 시간창 | 만드는 것 |
|---|---|---|---|
| 국장용(`--kr`) | 19:00(+22:00 재시도) | 16:00~다음날 08:00 | 검증 + 종목별 서술 + 시황 4문장까지 전부(**전일 한국장 기준으로 범위를 좁혀** 19시에 이미 확정) |
| 미장용(`--us`) | 09:30(+12:30 재시도) | 06:00~16:00 | 검증 + 종목별 서술 + 시황 4문장까지 전부(마감 확정이라 가능) |

- 캐시 파일: `output/pregen_kr.json` / `output/pregen_us.json` — `by_sym`(검증) +
  `written`(종목별 서술) + `sells_written` + `market_written`(시황 4문장, 국장·미장 둘 다).
- 발송 시점(`daily_ai_report.py`)이 이 캐시를 읽어 `for_kst`(대상일)가 오늘과 일치하면 그대로
  재사용. **캐시에 없는 심볼(후보풀이 살짝 바뀐 드문 경우)만** 지표+프로필 기반 무료 자동 문구
  (`_auto_fields`)로 채운다 — 이 경로는 API 호출이 전혀 없다.
- **2026-07-10 단순화**: 국장 메일의 시황 4문장은 원래 "밤사이 미국 마감까지 포함"을 노려
  19시엔 못 채우고 발송 시점(08시) 경량 API 콜로 메꿨는데, 그 콜은 haiku 기준 실제로는
  $0.001~0.003 수준(비용 문제는 아니었음) — 그래도 "전일 한국장 기준"으로 범위를 좁혀 19시
  pregen 시점에 이미 확정된 코스피·코스닥 데이터만으로 전부 끝내도록 단순화했다. 세계지수는
  국장 메일의 '전일 시장 요약' 표(코드 생성, §3.4)가 이미 보여주고 있어 중복도 아니다.
- 실패 사다리: 서술 생성만 실패 → 검증은 여전히 재사용(발송 시점 API 폴백 1콜, 단 아래 §4.4
  참고) / pregen 자체가 없음(PC 꺼짐) → 검증+서술 전부 API 폴백(발송은 됨).
- 재시도가 코드 변경 없이 가능한 이유: 메일 분리로 pregen~발송 사이 여유시간이 커졌다
  (국장 19:00→08:00 = 13시간, 미장 09:30→16:40 = 7시간10분). 시간창 가드가 이미 유효성을
  스스로 판단하므로 스케줄러 트리거만 하나 더 추가하면 그게 곧 재시도다.
- **`build_report`는 pregen 캐시(`by_sym`)만 있으면 API/CLI 가용 여부(`_enabled()`)와 무관하게
  동작한다**(2026-07-10 수정) — 이전엔 API 키가 없으면 pregen이 완벽해도 무조건 폴백해버리는
  버그가 있었다.

## 5. 자동 실행 흐름 (스케줄 전부)

### 5.1 GitHub Actions (`.github/workflows/report.yml`) — 실제 발송을 담당

| Cron(UTC) | KST 환산 | 모드 |
|---|---|---|
| `0 23 * * 0-4` | 월~금 08:00 | `--kr`(국장 장전) |
| `40 7 * * 2-6` | 화~토 16:40 실행(도착 ~17:00) | `--us`(미장 마감) |
| `30 22 * * 6` | 일 07:30 | `--weekly`(주간 자산배분) |

- 어느 cron이 깨웠는지(`github.event.schedule`)로 모드를 자동 분기.
- `workflow_dispatch`로 수동 실행 가능: `mode` 입력에 `kr`/`us`/`weekly`/`daily`(둘 다)/
  `preview`(발송 없이 미리보기만) 지정.
- 상태 영속화: 러너는 매 실행 초기화되므로, 실행 후 `output/ai_holdings.json`,
  `output/kr_holdings.json`, `output/last_sent.json`, `output/kospi200_cache.json`,
  `state_prev_list.json`을 저장소에 commit-back한다.
- Actions 환경변수(비용 관련): `AI_BACKEND=api`, `REPORT_MODEL_VERIFY=claude-sonnet-5`,
  `REPORT_MODEL_WRITE=claude-haiku-4-5`, `REPORT_WEB_USES=4`.
- 필요한 Secrets(총 6개): `SMTP_USER`, `SMTP_PASS`, `EMAIL_TO`, `ANTHROPIC_API_KEY`(폴백용),
  `KRX_ID`, `KRX_PW`(한국 데이터용 — 2025-12-27 KRX 정책 변경으로 신규 필수).

### 5.2 로컬 PC 작업 스케줄러 (`register_pregen_task.ps1`) — 비용을 줄이는 사전 작업

| 태스크 | 트리거 시각(KST) | 스크립트 |
|---|---|---|
| `StockPregenKR` | 19:00 + 22:00(재시도) | `run_pregen.ps1 -Mode kr` → `pregen.py --kr` |
| `StockPregenUS` | 09:30 + 12:30(재시도) | `run_pregen.ps1 -Mode us` → `pregen.py --us` |

- 둘 다 `StartWhenAvailable`(PC가 꺼져 있으면 다음 부팅 시 실행) + `AllowStartIfOnBatteries`.
- `run_pregen.ps1` 흐름: `git pull --rebase`(최신 상태파일 반영) → `pregen.py` 실행(AI_BACKEND=cli
  강제, 로컬 claude 구독 사용) → 성공 시 `output/pregen_{kr,us}.json`만 커밋·푸시. 실패해도 조용히
  종료(Actions가 API로 자동 폴백하므로 발송 지장 없음).
- 등록: 관리자 PowerShell에서 `.\register_pregen_task.ps1` 1회 실행.
- 수동 검토용: `.\test_pregen.ps1`(`-Mode kr`/`us`/`both`) — 실제 로컬 CLI로 pregen+리포트를
  돌려 **발송·git 조작 없이** `output/kr_report.html`/`us_report.html`을 만들어 브라우저로 연다.

### 5.3 중복 발송 방지 / 휴장일 처리

- `output/last_sent.json`의 `sent_kr_kst`/`sent_us_kst`(KST 날짜 문자열)로 국장/미장 메일을
  독립적으로 가드 — 같은 날 여러 번 실행돼도 실제 발송은 1회. 발송 **성공 시에만** 기록되므로
  실패한 날은 다음 실행에서 자동 재시도.
- 강제 재발송: `--force` 플래그(중복 가드 무시).
- 휴장일에도 발송은 계속되며(전 세계 지수·코인은 매일 갱신), 상단 배너로 "휴장 — 직전 거래일
  기준" 안내.

### 5.4 수동 실행 명령 모음

```bash
# 로컬 PC (claude CLI 사용, 무료 — set AI_BACKEND=cli 는 pregen.py가 내부에서 자동 설정)
python daily_ai_report.py --kr --no-email       # 국장 메일 미리보기만
python daily_ai_report.py --us --no-email       # 미장 메일 미리보기만
python daily_ai_report.py --daily --no-email    # 둘 다 미리보기
python daily_ai_report.py --weekly --no-email   # 주간 리포트 미리보기
python daily_ai_report.py --kr --force          # 실제 발송(중복가드 무시)

python pregen.py --kr                           # 국장용 사전생성 수동 실행
python pregen.py --us                           # 미장용 사전생성 수동 실행

python gen_profiles.py --limit 3 --dry-run      # 프로필 캐시 소규모 시험(파일 미변경)
python gen_profiles.py --refresh                # 프로필 캐시 전체 재생성(분기 1회 권장)

python backtest_weights.py --years 10 --keep 8 --oos 0.4   # 미국 팩터가중치 재검증
```

## 6. 구현 히스토리 (주요 마일스톤 요약)

1. **초기 버전**: S&P500 팩터 랭킹 + AI(API) 해설 단일 일일 메일. 회당 비용 ~$0.5로 지속 불가능한
   수준이었음.
2. **전략 개편**: `market_signals.py`(지수·코인 추세신호 엔진 신설), `kr_stocks.py`(코스피200 선별
   신설), 미국 진입필터·매도 트레일링 강화(-25%→-20%) — 근거는 STRATEGY.md 리서치 조사로 확정.
3. **AI 검증 레이어 도입**: 규칙이 후보 풀을 뽑고 AI가 웹검색으로 검증(매수유지/관찰강등/제외),
   코드가 최종 확정하는 3단계 구조 확립.
4. **상세화 + 비용 최소화 1차**: `entry_plan.py` 신설(분할매수·손절 100% 하드코딩), AI를
   검증(sonnet)/서술(haiku) 2단계로 분리, `gen_profiles.py`(프로필 캐시 Batch API), `pregen.py`
   1차 버전(검증만 사전생성) — 목표 pregen 있는 날 ~$0.02·없는 날 ~$0.12.
5. **메일 2통 분리**: 국장(장전)/미장(마감) 메일을 독립 스케줄·독립 pregen으로 분리(이전엔 시간대
   문제로 한쪽 데이터가 항상 불완전했음).
6. **완전 사전생성 + 재시도**: 메일 분리로 생긴 여유시간을 이용해 `pregen.py`가 서술까지 사전생성
   (`_write_ahead`), `build_report`가 사전서술 캐시를 우선 사용하도록 확장. 재시도 트리거 추가
   (코드 변경 없이 스케줄러 트리거만 추가). `gen_profiles.py`도 로컬 CLI 우선으로 전환.
7. **버그 수정 3건**(실사용 중 발견):
   - CLI 경로에 `--model` 인자가 아예 없어 검증/서술 모두 CLI 기본모델로 나가던 문제 → 수정.
   - `--allowedTools`는 도구를 사전승인만 할 뿐 다른 도구(Write)를 못 막아서, 서술 단계에서 모델이
     파일 저장을 시도하다 권한 대기 상태로 빠져 **AI 해설이 통째로 누락**되던 문제 → `--tools`로
     교체(공식 문서 확인 후 수정, 검증="WebSearch,WebFetch"만 / 서술=""로 도구 완전 비활성).
   - KRX가 2025-12-27부터 로그인 필수(KRX Data Marketplace)로 전환되며 pykrx가 멈춘 문제 →
     `KRX_ID`/`KRX_PW` 환경변수 지원 배선(로컬 영구 등록 + GitHub Secrets).
   - Windows PowerShell 5.1이 BOM 없는 UTF-8 스크립트의 한글을 콘솔에서 깨뜨리는 문제 → BOM +
     콘솔 인코딩 강제로 수정.
8. **콘텐츠 재배치**: 추세신호 카드를 메일별로 분리(국장=코스피·코스닥·금, 미장=나스닥100·
   S&P500·비트코인, 이더리움 제거), 전일 시장 요약을 국장 메일 전용으로 이동하고 자산 재정의
   (유럽=VGK, 글로벌=ACWI 등 대표 ETF로 통일, 전일 등락만 표시), 국장 카드·차트에서 종목코드를
   없애고 이름으로 대체.

## 9. 향후 개선 아이디어 (백테스트로 검증 후 채택 검토)

- 미국 팩터 가중치·IC는 분기~반기 단위로 감쇠하므로 `backtest_weights.py` 정기 재실행 필요.
- 한국 종목도 미국처럼 펀더멘탈 팩터(마진·발생액 등) IC 검증 후 랭킹에 반영할 가치가 있는지 조사.
- 금(GOLD/GLD)에 "equity"용 파라미터(200일선 등)를 그대로 재사용 중 — 원자재 특유의 사이클(코인처럼
  더 짧은 창이 나을지)이 있는지 백테스트로 확인 가치 있음.
- 리밸런싱 밴드(1.2×/0.8×)·방어 컷 기준(12개월 음수)은 Daryanani(2008) 등 일반 문헌 기준이므로,
  이 포트폴리오 구성(주식60/채권/금/코인) 특화 최적값으로 별도 그리드서치 여지 있음
  (`backtest_alloc.py` 참고 — 1,728조합 그리드서치 이미 존재, 결과는 `output/best_alloc.json`).
- pregen 재시도 2회(현재 시각당 1회 추가)로도 실패가 잦다면 3번째 트리거 추가를 고려.
